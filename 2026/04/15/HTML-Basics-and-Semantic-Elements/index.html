第1课：HTML 基础与语义化

在现代前端开发中，HTML 看似简单，却承载着网页的骨架与语义。无论你是使用 React、Vue 还是 Angular，最终输出的都是 HTML 结构。对 HTML 语义化的深入理解，直接影响页面的可访问性（a11y）、搜索引擎优化（SEO） 以及代码可维护性。本节课将从最基础的文档结构讲起，深入标签语义，并带你避开实际开发中最常见的陷阱。

1. 一个完整的 HTML 文档结构（Living Standard）

首先，让我们看一个符合最新 HTML Living Standard（WHATWG 维护）的现代 HTML5 文档模板。这个模板不是“随意写的”，每一个部分都有明确的规范要求。

原理性解释

元素/属性 作用与规范要求
<!DOCTYPE html> 声明文档类型为 HTML5。若不写或写错，浏览器将进入怪异模式（Quirks Mode），导致不同浏览器盒模型计算不一致，布局错乱。
<html lang="zh-CN"> 声明页面主要语言。这对屏幕阅读器的发音引擎至关重要，若为 en 而内容是中文，读屏软件会尝试用英文发音念中文。
<meta charset="UTF-8"> 必须在 head 的前 1024 字节内出现。未指定时浏览器可能猜测编码导致乱码。UTF-8 是全球通用的 Unicode 编码。
<meta name="viewport"> 移动端适配的基石。width=device-width 使视口宽度等于设备宽度；initial-scale=1.0 防止默认缩放导致字体过小。
<title> 必须唯一。它不仅是浏览器标签页文字，更是搜索引擎结果页的主要标题和收藏夹名称。
语义标签 (<header>, <main>, <article>, <section>, <footer>) 这些是 HTML5 引入的分段内容元素。它们帮助构建文档大纲（Document Outline），辅助工具据此生成页面导航。

2. 核心语义化标签详解：不仅仅是 DIV

在 HTML5 之前，开发者只能使用 <div class="header"> 来表达头部。现在我们有了一套丰富的语义标签。理解它们的适用场景是避免滥用的关键。

2.1 <header> vs <h1>～<h6>

  • **<header>**：介绍性内容或导航链接的容器。一个页面可以有多个 <header>（例如全局头部和文章头部）。
  • <h1>：当前页面或当前分段内容（Sectioning Content） 的最高级标题。HTML5 大纲算法曾允许多个 <h1>，但实践中强烈建议一个页面只用一个 <h1> 作为核心主题，以确保 SEO 和可访问性工具的最优理解。

2.2 <section> vs <article> 的经典混淆

这是面试和代码审查中的高频易错点。

对比维度 <section> <article>
语义定义 文档或应用中一个主题分组。通常需要包含标题（h1-h6）。 独立、可分发或可重用的内容块。例如一篇博客帖子、一条评论、一个天气小部件。
典型场景 将长文章分为“第一章”、“第二章”；或者将首页分为“热门商品”、“新闻列表”区块。 博客文章列表中的单篇文章、论坛中的单个帖子、RSS 订阅中的条目。
独立发布 脱离上下文通常意义不完整。 脱离上下文依然保持完整意义。例如，把一篇 <article> 放到另一个页面依然通顺。
使用建议 如果仅仅是为了样式分组或JS 挂载钩子，请使用 <div>，不要滥用 <section>。 当内容包含 <header>、<footer> 和自己的标题时，优先考虑 <article>。

2.3 <figure> 与 <figcaption>：图表与配文的正确搭配

当展示图片、图表、代码块、引文时，若需要关联说明文字，使用这对组合可替代 <div class="image-wrapper">。

原理：搜索引擎和辅助工具会将 <figcaption> 的内容与 <img> 的 alt 属性结合理解，提升内容的可发现性。loading="lazy" 是现代浏览器原生支持的懒加载属性，无需 JS 即可优化性能。

2.4 交互元素 <details> 与 <summary>

这是一个内置的可折叠/展开组件，完全无需 JavaScript。

3. 表单语义化与无障碍增强（ARIA 初探）

表单是最具交互性的 HTML 部分，也是无障碍问题的重灾区。

3.1 显式关联 <label>

唯一正确的做法：使用 for 属性指向输入框的 id。

3.2 <fieldset> 与 <legend> 分组

用于将相关控件分组，例如注册表单中的“个人信息”和“账户信息”。

原理：屏幕阅读器在进入 <fieldset> 时会读出 <legend> 的内容，告知用户当前组的作用。视觉上也提供了清晰的边界（可通过 CSS 重置，但语义保留）。

3.3 输入类型与原生验证

HTML5 引入了多种输入类型，浏览器会提供内置的键盘优化和验证反馈。

类型 移动端键盘 浏览器自动验证规则
type="email" 英文键盘带 @ 和 . 必须包含 @ 且无非法空格
type="tel" 数字拨号键盘 无特定格式限制，用于电话号码输入，防止浏览器自动纠错
type="url" 英文键盘带 . 斜杠 必须是有效 URL 协议格式（如 https:// 开头）
type="number" 数字键盘 仅允许数字、e（科学计数法）和部分符号，提交时验证
type="search" 搜索/回车键 通常带有清除按钮样式（浏览器实现）

易错点：不要使用 type="number" 来输入电话号码或身份证号。因为 number 输入框在输入过大的数字时会被转为科学计数法（如 1e+10），且无法输入空格、括号、连字符。

4. HTML 在框架中的角色：React、Vue、Angular 中的 JSX 与模板

当你写 React 的 return <div>Hello</div> 时，这并不是真正的 HTML，而是 JSX。理解这种差异对调试至关重要。

方面 原生 HTML React JSX Vue 模板 / Angular 模板
语法解析器 浏览器 HTML 解析器 Babel / TypeScript 编译器 Vue 编译器 / Angular 编译器
属性命名 class, for, tabindex className, htmlFor, tabIndex (camelCase) class, for (与 HTML 一致，因为模板更接近 HTML 方言)
布尔属性写法 <input disabled> 或 disabled="" 或 disabled="disabled" disabled={true} 或直接 disabled（值必须为 JS 表达式） <input disabled> (与 HTML 一致)
样式对象 字符串 <div style="color: red"> 对象 style={{ color: 'red' }} 字符串或对象绑定 :style="{ color: 'red' }"
注释 <!-- 注释 --> {/* 注释 */} <!-- 注释 -->

提示词工程关联：当向 AI 描述一个 UI 需求时，明确指定框架可避免生成的代码带有错误的属性名。例如，提示词应写为：

“请使用 React 和 TypeScript，生成一个包含用户名输入框和提交按钮的表单。使用 useState 管理输入值，并注意 JSX 属性应为 className 和 htmlFor。”

二、AI 编程任务：编写面向 AI 的提示词

场景：你需要快速搭建一个个人简历卡片的 HTML 结构，要求如下：

  • 必须包含头像区域、姓名、职位、一段个人简介。
  • 必须包含联系信息列表（邮箱、电话、地址）。
  • 必须使用语义化标签，且头像图片需支持懒加载。
  • 不使用任何 CSS，仅关注 HTML 结构干净、可访问性好。

任务要求：请你写出一段完整的中文提示词，发送给 ChatGPT 或 Copilot，使其能够直接生成符合上述要求的 HTML 代码。提示词中应明确指出框架要求（原生 HTML）、语义化要求及具体功能点。

💡 提示：一个好的提示词包含角色设定、输出格式、具体元素列表、约束条件。

三、面试真题与参考答案

题目（字节跳动前端面试题）：

请详细阐述 <script> 标签的 async 和 defer 属性的区别，并结合浏览器解析 HTML 的过程画出加载执行时序图。此外，解释为什么将 <script> 放在 <body> 底部是一种常见的性能优化实践。

参考答案：

  1. 区别与原理：

    • 普通脚本（无属性）：浏览器遇到 <script> 时，停止 HTML 解析，立即下载（若外链）并执行脚本。执行完毕后才继续解析 HTML。这会导致页面阻塞渲染。
    • defer：脚本异步下载，不阻塞 HTML 解析。脚本会在 HTML 解析完成后、DOMContentLoaded 事件之前按在文档中的顺序依次执行。适用于依赖 DOM 完整性的非关键脚本。
    • async：脚本异步下载，下载完成后立即暂停 HTML 解析并执行脚本。执行顺序不固定，谁先下载完谁先执行。适用于完全独立的第三方统计、广告脚本。

  2. 时序图（文字描述）：

    • 普通：HTML解析 -> [遇到script] -> 暂停解析 -> 下载+执行 -> 恢复解析 -> DOMContentLoaded。
    • defer：HTML解析(同时异步下载) -> 解析完成 -> 执行defer脚本(按顺序) -> DOMContentLoaded。
    • async：HTML解析(同时异步下载) -> 下载完成立即暂停解析 -> 执行async脚本 -> 恢复解析 -> DOMContentLoaded（此时可能已执行部分async）。

  3. 放在 <body> 底部的原因：

    • 将 <script> 放在 </body> 之前，浏览器会优先解析和渲染 HTML/CSS，让用户尽快看到页面内容（首屏渲染）。当 DOM 树基本构建完毕后，再去加载和执行 JS，可避免长时间的白屏。
    • 对于旧代码或不支持 defer 的老浏览器，这是确保非阻塞渲染的兼容性最佳实践。在现代工程中，推荐使用 <script defer> 放在 <head> 中，既能提前下载，又能不阻塞渲染。

课后练习答案

一、概念自测答案

  1. C

    • 解析：<article> 标签专门用于表示独立、可重用的内容，如博客文章、新闻条目。

  2. C

    • 解析：<meta charset="UTF-8"> 必须出现在 <head> 的前 1024 字节内，放在 <body> 中将无效，浏览器在解析 body 时已经确定了编码，此时指定已晚。

  3. for

    • 解析：<label for="pwd"> 将标签与 id="pwd" 的输入框关联，点击标签即可聚焦输入框，提升可用性。

  4. A、B、D

    • 解析：<header>、<nav>、<footer> 均为 HTML5 新增的语义化标签，<div> 和 <span> 是无语义的通用容器，不属于 HTML5 新增。

示例提示词：
“你是一名资深前端开发，请使用原生 HTML5 为我生成一个个人简历卡片的完整结构。要求：

  4. 整体结构需用 <main> 包裹，不使用任何 CSS 或内联样式，仅输出纯语义化 HTML。
  5. 所有交互元素（如链接邮箱）需正确使用 <a href="mailto:...">。请直接输出代码。”
